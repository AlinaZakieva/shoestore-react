# MVP проекта «ShoeStore React»

## 1. Цель MVP

MVP проекта «ShoeStore React» — это минимально работоспособная версия интернет-магазина обуви, которая позволяет пользователю:

- просматривать каталог обуви;
- открывать карточку конкретного товара;
- добавлять товары в корзину и удалять их оттуда;
- видеть текущий состав корзины и итоговую стоимость.

На этом этапе не реализуются личный кабинет, полноценное оформление заказа и сложная интеграция с внешними системами. Основной акцент делается на пользовательском интерфейсе и базовых сценариях работы с каталогом и корзиной.

---

## 2. Роль пользователя в MVP

В рамках MVP рассматривается одна основная роль:

- **Покупатель (гость)** — анонимный пользователь, который:
  - заходит на сайт без регистрации;
  - просматривает каталог обуви;
  - открывает карточки товаров;
  - добавляет товары в корзину;
  - управляет содержимым корзины.

Авторизация и личный кабинет могут быть добавлены на последующих этапах разработки, но не являются частью минимального набора требований.

---

## 3. Функциональные требования к MVP

### 3.1. Каталог товаров

1. Пользователь должен видеть страницу каталога обуви по адресу `http://localhost:5173`.
2. Каталог отображает список товаров в виде адаптивной сетки карточек.
3. Каждая карточка товара в каталоге содержит:
   - фотографию обуви;
   - название модели;
   - краткое описание;
   - цену;
   - кнопку «Добавить в корзину».
4. При клике по карточке товара пользователь переходит на страницу карточки товара.

### 3.2. Карточка товара

1. Пользователь должен иметь возможность открыть карточку товара.
2. На странице карточки товара отображаются:
   - фотография обуви;
   - название модели;
   - полное описание;
   - основные характеристики (например, материал, тип, размерный ряд);
   - цена;
   - кнопка «Добавить в корзину».
3. На странице карточки товара должна быть кнопка или ссылка для возврата в каталог.
4. Добавление товара в корзину из карточки должно изменять состояние корзины (количество и сумма).

### 3.3. Корзина

1. Пользователь должен иметь доступ к странице корзины.
2. На странице корзины отображаются:
   - список добавленных товаров;
   - название каждой модели;
   - цена позиции;
   - итоговая стоимость всех товаров.
3. Пользователь должен иметь возможность:
   - удалить товар из корзины;
   - очистить корзину полностью (опционально, но желательно).
4. При удалении товаров отображаемая сумма и количество позиций пересчитываются.

### 3.4. Навигация и общий интерфейс

1. На всех основных страницах (каталог, карточка товара, корзина) отображается шапка сайта с:
   - логотипом магазина;
   - ссылкой на каталог;
   - мини-корзиной (общее количество товаров и сумма).
2. Клик по логотипу переводит пользователя на главную страницу (каталог).
3. Клик по мини-корзине в шапке переводит пользователя на страницу корзины.
4. Для неизвестных адресов отображается страница 404 с возможностью вернуться на главную страницу.

## 4. Нефункциональные требования к MVP

1. **Адаптивность интерфейса.**  
   Интерфейс должен корректно отображаться на настольных компьютерах и ноутбуках. Базовая поддержка планшетов и телефонов приветствуется, но не является обязательной.

2. **Технологический стек.**  
   - React и TypeScript для разработки интерфейса.
   - React Router для организации маршрутизации.
   - Собственная библиотека компонентов `ui-library` для карточек товаров, сетки и базовых элементов интерфейса.

3. **Качество кода.**  
   - использование ESLint и Prettier;
   - разбиение проекта на модули согласно архитектуре (`apps/web`, `packages/ui-library`).

4. **Тестирование.**  
   - наличие модульных/компонентных тестов для ключевых частей интерфейса (например, карточка товара, каталог, базовая логика корзины);
   - использование Jest и React Testing Library.

## 5. Основной use case: выбор обуви и формирование корзины

### 5.1. Краткое описание

Покупатель заходит на сайт, просматривает каталог, выбирает модель обуви, открывает карточку товара, добавляет товар в корзину и проверяет содержимое корзины.

### 5.2. Участники

- **Основной участник:** Покупатель (гость).
- **Система:** Веб-приложение «ShoeStore React».

### 5.3. Предусловия

- Сайт доступен по URL.
- Клиентское приложение загружается без ошибок.
- В системе есть хотя бы один товар для отображения в каталоге.

### 5.4. Основной сценарий

1. Покупатель открывает главную страницу сайта.  
   Система отображает каталог обуви в виде сетки карточек товаров.

2. Покупатель просматривает список товаров и выбирает интересующую модель.  
   Система показывает на карточках название, цену и фотографию.

3. Покупатель кликает по карточке выбранного товара.  
   Система открывает страницу карточки товара с подробной информацией.

4. Покупатель изучает описание и характеристики товара.  
   Система отображает полное описание, характеристики и цену.

5. Покупатель нажимает кнопку «Добавить в корзину» на странице карточки товара.  
   Система:
   - добавляет выбранный товар в корзину;
   - обновляет количество товаров и сумму в мини-корзине в шапке.

6. Покупатель нажимает на мини-корзину или кнопку перехода в корзину.  
   Система открывает страницу с содержимым корзины.

7. Покупатель проверяет список товаров в корзине и итоговую сумму.  
   Система отображает:
   - перечень добавленных моделей;
   - цену каждой позиции;
   - общую стоимость.

8. При необходимости покупатель удаляет ненужную позицию.  
   Система удаляет товар из корзины и пересчитывает итоговую сумму и количество товаров.

Результат: покупатель получает сформированную корзину с выбранной обувью и корректной суммой заказа.

### 5.5. Альтернативные сценарии

- **Альтернатива 1: добавление товара из каталога.**  
  Вместо перехода на карточку товара покупатель может добавить модель в корзину прямо из каталога, нажав кнопку «Добавить в корзину» на карточке.  
  В этом случае система так же обновляет мини-корзину и позволяет перейти на страницу `/cart`.

- **Альтернатива 2: пустая корзина.**  
  Если покупатель открывает страницу `/cart`, не добавив ни одного товара, система отображает сообщение о том, что корзина пуста, и предлагает вернуться в каталог.

---

Таким образом, MVP проекта «ShoeStore React» обеспечивает базовый, но завершенный пользовательский сценарий: от просмотра каталога до формирования корзины с выбранной обувью. Этого достаточно, чтобы продемонстрировать работоспособность архитектуры, библиотеку компонентов и основной функционал интернет-магазина.
